# 하네스 엔지니어링 초기 셋업 — 웹서비스 (Next.js 풀스택)

> **스택**: Next.js(프론트+백) · TypeScript · Tailwind v4 + shadcn/ui · Supabase(Auth+Postgres) · Prisma · Docker Compose(OCI Ubuntu) · GitHub Actions → SSH 배포

## 이 문서의 성격 (먼저 읽을 것)

이 문서는 하네스 **골격 스펙**이다. OpenAI Harness Engineering 원문의 *개념*에만 충실하며, 다음을 **하지 않는다**:

- **도구별 세부 구현을 기술하지 않는다** — 연결 문자열, 설정 문법, Dockerfile 내용, 정확한 패키지명. 이건 가장 빨리 낡는 부분이라 골격에 두면 드리프트의 진원지가 된다. 에이전트가 빌드 시점에 *현재 공식 문서를 확인해* `docs/`에 직접 작성한다.
- **설계 결정을 미리 정하지 않는다** — 보안 모델(RLS냐 앱 레이어 인가냐), 아키텍처 레이어 구성, 구체적 불변식 목록. 이것들은 스택·요구사항·운영에 따라 달라지고, 원문대로 *시간이 지나며 인코딩*된다. 골격은 이런 결정이 **들어갈 슬롯과 그것을 강제할 기계 장치**만 만든다.

> 원문 근거: "이런 동작은 특정 구조·툴링에 크게 의존하므로 일반화 가정 금지" / "문서화가 부족하면 규칙을 코드로 승격" / 불변식은 반복 하드닝으로 누적된다.

---

## 0. 메타 원칙 (원문 6개념 — 모든 작업에 적용)

1. **맵, 매뉴얼 아님** — CLAUDE.md는 짧은 지도. 상세·결정은 docs/로. 컨텍스트는 희소 자원.
2. **불변식은 기계 강제, 구현은 자유** — 금지는 훅·린트·권한으로. 가이드는 docs로. "말로만 금지" 금지.
3. **에이전트 가독성** — 컨텍스트에서 못 보면 존재하지 않는 것. 암묵지는 repo에 마크다운으로 인코딩.
4. **검증은 컨텍스트 효율적으로** — 성공은 침묵, 실패만 자세히.
5. **고위험엔 사람** — 중앙에서 경계, 로컬에서 자율. 배포·마이그레이션·파괴적 작업은 사람 게이트.
6. **반복 하드닝** — 에이전트가 막히면 신호로 보고 메커니즘(도구/가드레일/문서)을 하나 추가, `lessons.md`에 기록.

**작업 방식**: 도구별 최신 설정은 추측하지 말고 공식 문서/Context7로 확인 후 적용. 설계 결정은 미리 못박지 말고 ADR로 기록.

**Step 0 (스캐폴딩)**: 베이스 스택과 의존성은 에이전트가 현재 CLI로 생성·설치한 뒤, 그 위에 아래 하네스를 얹는다.

> **Claude Code 동작 전제**: 권한 우선순위 deny>ask>allow · 권한 문법 `Bash(<cmd>:*)` · `cat`은 읽기전용 내장이라 `.env`는 권한만으론 못 막음 → **훅이 1차 방어** · deny는 트립와이어(우회 가능) → 훅 보강 · 훅은 세션 시작 시 스냅샷됨 → 새로 만든 가드는 **다음 세션부터** 작동.

## 1. Memory — `CLAUDE.md` (맵)

- **≤150줄**, 컨텍스트 압축에서 살아남는 순수 라우팅.
- 맨 위: "무엇이 어디 있는지" 라우팅 표 + `docs/` 인덱스 + 세션 진입점 한 줄.
- **불변식 섹션**: 초기엔 **비어 있거나 최소**. 결정이 내려질 때마다 한 줄씩 추가하며, 각 항목은 **(왜 → `docs/adr/NNN`)** 와 **(강제 → 린트 규칙명/훅 파일)** 을 가리킨다.
  > 원문: 불변식은 처음부터 다 정하지 않는다. 학습하며 인코딩된다.

## 2. Permissions — `.claude/settings.json` (고위험엔 사람)

명령을 **카테고리로** 규정한다. 정확한 명령·문법은 에이전트가 현재 스택에 맞게 완성한다.

- **allow** — 읽기·검증성 작업 (파일 읽기/검색, 읽기성 git, typecheck/lint/test/build, 포맷터).
- **ask** — 고위험, 사람 확인: 배포 명령, DB 마이그레이션 적용, 컨테이너 기동/정지, 원격 푸시.
- **deny** — 트립와이어: 파괴적 명령, 비밀 파일(`.env*`) 읽기/쓰기, 빌드 산출물 편집.

## 3. Hooks (기계 강제 — *동작*만 규정, 대상 목록은 docs 바인딩)

- **PreToolUse (가드)**: ① 비밀 파일 접근 차단(읽기·편집·스테이징, `cat` 포함) ② 빌드 산출물 편집 차단 ③ 파괴적 명령 차단. 차단 시 exit 2 + 사유·대안 출력.
- **PostToolUse**: 변경 파일만 포맷+lint. **성공 침묵, 실패만 출력.** 무거운 검사는 여기 두지 않음.
- **SessionStart**: 최근 git log + `HANDOFF.md` + 진행 중 exec-plan + `lessons.md`를 주입(기록 시스템 + 연속성).
- **Stop**: `HANDOFF.md` 갱신(요약/남은 TODO/다음 진입점) + 필요 시 `lessons.md` 추가.
- **pre-commit (Lefthook)**: typecheck · 전체 lint(경계 포함) · 관련 test.
- **피드백 사다리**: PostToolUse(ms) → pre-commit(s) → CI(min) → 사람(h). 빠른 레이어일수록 컨텍스트가 깨끗하다.

## 4. Commands & Skills (이름·트리거·목적만; 절차는 각 파일에 에이전트가 작성)

- **commands**(사용자 호출): `/commit`(검증 후 컨벤셔널 커밋) · `/review`(diff 셀프리뷰: 타입/린트/불변식/정의완료) · `/deploy`(**고위험 — §2 ask + §3 훅으로 기계 게이트**) · `/cleanup`(엔트로피 GC: 죽은 코드·미사용 의존성·docs 신선도).
- **skills**(자동 활성화, description이 트리거): 기능 수직 슬라이스 스캐폴딩 · UI 컴포넌트 추가 규칙.
  > 원문: 개발 루프(테스트·검증·리뷰·복구)를 시스템에 인코딩.

## 5. `docs/` — 기록 시스템 (슬롯; **내용은 개발 중 작성**)

각 문서는 "다뤄야 할 주제"만 정해두고, 결정·상세는 개발하며 채운다.

- `architecture.md` — 아키텍처 맵·경계. *구체 레이어 구성은 ADR로 결정·기록* (원문의 레이어드 패턴은 채택 후보).
- `design-system.md` — UI 토큰·컴포넌트 컨벤션 슬롯.
- `data.md` — 데이터 접근·마이그레이션 절차 슬롯.
- `security.md` — 인증/인가 경계·비밀 취급 **모델 슬롯**. ⚠️ 구체 모델(RLS냐 앱 레이어냐 등)은 *미리 정하지 않고* ADR로 기록.
- `state.md` — 상태 관리 범위 슬롯.
- `verification.md` — 검증 경로(단위·E2E·런타임 관찰) 슬롯. *원문: 앱을 에이전트가 읽을 수 있게.*
- `deploy.md` — 배포 흐름·롤백 슬롯.
- `adr/` — 모든 주요 결정의 "왜"(보안·아키텍처 등). 초기엔 ADR 템플릿만.
- `exec-plans/{active,completed,tech-debt}` · `HANDOFF.md` · `lessons.md`
  > 원문: docs는 기록 시스템 · 계획은 일급 아티팩트 · 린터/CI로 신선도·교차링크 검증 · doc-gardening으로 드리프트 정리.

## 6. 아키텍처/취향 강제 (원문 핵심 — *메커니즘*만)

- 경계 린트(boundary lint)로 의존성 방향·교차 경계를 강제할 **틀**을 깐다. *구체 규칙은 ADR로 결정되면 추가.*
- env 검증으로 빠른 실패(loud fail).
- 린트는 커스텀이므로 **에러 메시지에 수정 지침을 주입**해 에이전트 컨텍스트로 전달.
  > 원문: 맞춤 린터+구조 테스트로 불변식 기계 강제. 중앙에서 경계, 로컬에서 자율.

## 7. 엔트로피 & 가비지 컬렉션 (원문)

- "황금 원칙"과 정기 정리 프로세스의 **슬롯**. 초기엔 `/cleanup` + `lessons.md`로 시작, 규모가 커지면 백그라운드 doc-gardening/리팩터 작업으로 확장.
  > 원문: 기술 부채는 한꺼번에 갚지 말고 매일 조금씩. 나쁜 패턴이 퍼지기 전에 매일 포착.

## 8. 산출물 체크리스트

- [ ] `CLAUDE.md`(맵, 불변식 슬롯) · `.claude/settings.json`(권한+훅) · `scripts/hooks/*`(실행권한)
- [ ] `.lefthook.yml` · `.gitignore`(`.env*`) · 경계 린트 틀
- [ ] `.claude/commands/*` · `.claude/skills/*`
- [ ] `docs/` 골격(슬롯) + ADR 템플릿 + 빈 `HANDOFF.md`/`lessons.md`
- [ ] **검증(새 세션)**: 비밀 파일 접근·파괴적 명령·산출물 편집을 *의도적으로 시도* → 훅이 실제로 차단하는지 확인 후 완료 처리.

> 적어둔 방어와 동작하는 방어는 다르다. 가드가 막는 걸 눈으로 확인한 뒤에야 셋업 완료다.
