# Requirements — 기능·비기능 요건 (검증 가능하게)

> **슬롯.** 기능요건은 유저스토리 + 수용기준, 비기능요건은 카테고리별로. 작성 컨벤션은 `requirements-discovery` 스킬.
> 각 Must-have 수용기준은 테스트로 직결된다 → `docs/verification.md`.

## 기능 요건 (유저스토리 + 수용기준)

### FR-001: <제목>  · 우선순위: Must
- **스토리**: <역할>로서 <목표>를 원한다, 그래야 <가치>.
- **수용기준**:
  - Given <상황>, When <행동>, Then <기대 결과>.

_(필요한 만큼 FR-NNN 추가. Must 부터. 시스템 수준 규칙은 EARS 표기 사용 가능 — 스킬 참고.)_

## 비기능 요건 (NFR)
- **성능/확장**: _목표 부하·응답시간._
- **보안/프라이버시**: _인증·인가·개인정보 (→ `docs/security.md` ADR)._
- **가용성**: _목표 가동률·장애 허용._
- **접근성(a11y)**: _준수 기준._
- **관측성**: _로그·메트릭·추적._
- **규정/컴플라이언스**: _해당 규제._
