> ⚠️ **고위험 작업.** 배포·원격 명령은 권한 `ask`(§2)와 가드 훅(§3)으로 기계 게이트된다.
> 이 명령은 사람의 명시적 확인 없이 원격에 무엇도 적용하지 않는다.

절차(구체 흐름은 `docs/deploy.md` 에 결정·기록된 것을 따른다):

1. **사전 점검** — 배포 대상 브랜치가 검증을 통과했는지(typecheck/lint/test/build) 확인. 실패 시 중단.
2. **마이그레이션 확인** — 적용할 DB 마이그레이션이 있으면 사람에게 명시적으로 알리고 승인받는다(`ask`).
3. **배포 실행** — `docs/deploy.md` 의 흐름(예: SSH/Docker Compose)을 따른다. 각 원격·컨테이너 명령은 권한 `ask` 로 사람이 승인.
4. **검증** — 배포 후 헬스체크/스모크 테스트(`docs/verification.md`).
5. **롤백 준비** — 실패 시 `docs/deploy.md` 의 롤백 절차. 롤백 지점을 미리 확인.

배포 흐름이 docs에 아직 없으면, 추측해서 실행하지 말고 **먼저 `docs/deploy.md` 를 채우고 ADR로 결정을 기록**한다.
